4)
Front-End Developers:
Responsible for making the application easy to understand and visually appealing. They design the user interface so important features are easy to find and use, create forms and buttons, and ensure the application works well on different devices and browsers.

Back-End Developers:
Responsible for the application's logic and functionality. They process user requests, validate information, manage user accounts, assign reports to the correct department, enforce business rules, and make sure the application works correctly behind the scenes.

Database Administrators (DBAs) / Data Engineers:
Manage the application's data. They design and maintain the database, ensure reports and user information are stored accurately, improve database performance, create backups, and help protect sensitive information from being lost or accessed by unauthorized users.

QA (Quality Assurance) Testers:
Test the application before updates are released. They look for bugs, verify that features work as expected, test different scenarios (such as invalid logins or incomplete reports), and ensure changes don't break existing functionality.

DevOps Engineers:
Keep the application running reliably. They deploy updates, monitor servers, improve security, manage cloud infrastructure, perform backups, and respond quickly if the application experiences outages or performance issues.

Product Managers and Designers:
Product managers decide which features should be built based on user and business needs. Designers plan the layout, appearance, and overall user experience to make the application intuitive and efficient.

Connection to My Experience:
My experience has involved problem-solving, troubleshooting, and helping users work through technical issues. Those skills relate closely to quality assurance and product design because they require understanding user needs, identifying problems, and finding practical solutions.
